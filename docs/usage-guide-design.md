# YakYak 使用引导设计

本文档定义 YakYak（说词儿啊！）的产品内使用引导。目标是让第一次打开页面的主播在 30 秒内理解核心工作流，并且不打断直播中的使用节奏。

## 设计目标

- 首次进入时，让用户知道 YakYak 的三个核心动作：看倒计时、说万能句、滑动完成流程词。
- 引导必须优先适配手机窄屏，不产生页面级滚动，不遮挡正在使用的主界面。
- 引导不能像说明书。它应该短、直接、可跳过，并且能在用户操作后自然消失。
- 引导需要覆盖隐藏交互：点击倒计时设置秒数、左右滑动完成、编辑词库、导入导出、长按拖动排序。
- 引导状态需要持久化到浏览器数据库，避免用户每次打开都重新看一遍。

## 引导策略

YakYak 不使用一次性长教程，而采用“首次总览 + 局部提示 + 可重新打开”的结构。产品内引导使用 `driver.js` 承载元素高亮、遮罩、popover 定位和步骤控制，避免自维护复杂的浮层定位逻辑。

1. 首次总览：第一次打开应用时出现 5 步轻量引导。
2. 局部提示：用户进入特定区域时出现贴近控件的短提示。
3. 帮助入口：顶部工具区增加一个帮助按钮，用户可以重新查看引导。
4. 自动消隐：当用户完成对应操作后，该提示不再显示。

## 信息架构

### 入口

顶部工具区增加一个帮助按钮，建议放在导入/导出按钮左侧或编辑按钮左侧。图标使用 `CircleHelp`，点击后打开“使用引导”弹窗。

按钮规则：

- 图标按钮，不显示文字。
- `aria-label="使用引导"`。
- light/dark 样式跟随现有顶部工具按钮。
- 首次未完成引导时，可以加一个很轻的青色提示点；完成后不显示。

### 首次引导弹窗

首次打开应用且 `guide.completed !== true` 时展示。

形式：

- 使用 `driver.js` popover 指向真实页面元素。
- 手机端 popover 控制在视口内，不触发页面级滚动。
- 背景使用 `driver.js` overlay，并用 YakYak 主题样式覆盖默认视觉。
- 弹窗内不超过 5 页，每页只讲一个动作。
- 底部有进度点、上一步、下一步/开始使用。
- 右上角关闭按钮用于退出本次引导。
- 首次自动展示前即标记 `guide.completed = true`，确保引导只自动出现一次。

文案：

第 1 页：说话节奏

- 标题：`盯住倒计时`
- 正文：`圆环会不断循环。进入最后三分之一时，下面会出现一句万能句，提醒你及时接话。`
- 辅助动作：`点倒计时可以改秒数。`

第 2 页：流程词

- 标题：`按流程往下讲`
- 正文：`最上面的流程词会高亮。讲完一条，左右滑动即可标记为已完成。`
- 辅助动作：`点重来可以恢复本轮流程。`

第 3 页：备份词库

- 标题：`备份你的词库`
- 正文：`导出 JSON 可以备份词库；换设备时再导入，就能继续使用。`
- 辅助动作：`导入导出按钮放在顶部工具区。`

第 4 页：编辑词库

- 标题：`把它改成你的话`
- 正文：`在编辑词库里添加流程词和万能句。流程词长按拖动排序，左右滑动删除。`
- 辅助动作：`编辑弹窗内会有一次单独的编辑引导。`

第 5 页：帮助入口

- 标题：`随时重新查看`
- 正文：`顶部问号按钮可以重新打开使用引导。`
- 辅助动作：`首次引导完成后提示点消失。`

## 局部提示

局部提示不是弹窗，而是贴近控件的小气泡。它们只在首次相关场景出现，并在用户操作后消失。

### 倒计时提示

触发条件：

- 首次进入应用后，用户还没有点过倒计时设置。
- 首次总览已关闭或完成。

展示位置：

- 倒计时圆环下方或右侧，不能遮挡万能句。
- 手机窄屏优先放在圆环下方。

文案：

`点这里调整说话间隔`

消失条件：

- 用户点击倒计时。
- 用户点击提示关闭。
- 展示超过 8 秒后自动消失，但不标记完成；下次仍可再出现，最多出现 3 次。

### 流程词滑动提示

触发条件：

- activeFlow 数量大于 0。
- 用户还没有完成过任何流程词。
- 首次总览已关闭或完成。

展示位置：

- 第一条流程词内部右侧，使用一个轻量 pill。

文案：

`左右滑动标记已完成`

消失条件：

- 用户完成任意流程词。
- 用户点击关闭。

### 编辑词库提示

触发条件：

- 用户第一次打开编辑词库。

展示位置：

- 编辑弹窗顶部，tab 下方。

文案：

`流程词可长按排序；左右滑动删除。万能句会在倒计时变红时随机出现。`

消失条件：

- 用户关闭编辑弹窗。
- 用户执行新增、删除或排序任意操作。

### 导入导出提示

触发条件：

- 用户第一次打开编辑词库，且从未导入/导出过。
- 或用户点击帮助按钮后选择“数据备份”。

展示位置：

- 顶部导入/导出图标附近。

文案：

`导出 JSON 备份词库，换设备时再导入。`

消失条件：

- 用户导入或导出成功。
- 用户关闭提示。

## 帮助弹窗设计

帮助按钮打开的是可重复查看的完整引导弹窗，不影响首次引导状态。

内容结构：

1. `直播时怎么用`
   - 看倒计时。
   - 红色阶段说万能句。
   - 讲完流程词就滑动完成。
2. `怎么配置`
   - 点倒计时改秒数。
   - 点编辑词库增删内容。
   - 流程词长按排序。
3. `怎么备份`
   - 导出 JSON。
   - 导入 YakYak JSON。
   - 数据默认保存在当前浏览器。

底部按钮：

- 主按钮：`知道了`
- 次按钮：`重新显示新手提示`

点击 `重新显示新手提示` 后：

- `guide.completed = false`
- 清空局部提示的已完成状态
- 立即打开首次引导第 1 页

## 状态持久化

建议在现有 IndexedDB snapshot 中新增 `guide` 字段。

```ts
type GuideState = {
  completed: boolean
  timerHintSeen: boolean
  flowSwipeHintSeen: boolean
  editorHintSeen: boolean
  importExportHintSeen: boolean
  timerHintDismissCount: number
}
```

兼容规则：

- 旧数据没有 `guide` 字段时，按未完成处理。
- 导入旧版 JSON 时自动补齐默认 `guide`。
- 导出时包含 `guide` 状态，但导入时不强制覆盖用户本机的引导状态。建议导入词库后保留当前设备的 guide 状态。

默认值：

```ts
const defaultGuideState: GuideState = {
  completed: false,
  timerHintSeen: false,
  flowSwipeHintSeen: false,
  editorHintSeen: false,
  importExportHintSeen: false,
  timerHintDismissCount: 0,
}
```

## 动效方案

首次引导弹窗：

- backdrop 使用现有 fade。
- panel 使用 `fly`，手机端从底部进入，桌面端轻微上浮进入。
- 页面切换使用横向 12px 位移 + 透明度变化，时长 160ms。

局部提示：

- 首次出现：opacity 0 -> 1，y 4px -> 0。
- 不使用循环跳动，避免直播场景中干扰视线。
- 可以使用一次性的轻微 scale 动效提示可交互。

流程词滑动提示：

- 可在第一条流程词上做一次短暂的水平示意动画，但不能真的改变数据。
- 动画只在首次提示出现后播放一次。

## 视觉规范

- 继续保持黑色直播工作台风格。
- 引导使用 cyan 作为主强调色，green 只用于成功/完成语义。
- 引导文字使用短句，不使用大段解释。
- 手机端弹窗宽度占满，左右留 12px。
- 弹窗圆角不超过现有 modal 规范。
- 不使用卡片套卡片。

## 可访问性

- 弹窗需要 `role="dialog"`、`aria-modal="true"`、标题绑定 `aria-labelledby`。
- 帮助按钮、关闭按钮、上一步、下一步都需要明确 `aria-label` 或文字。
- 局部提示不能只依赖颜色表达状态。
- prefers-reduced-motion 下关闭页面切换位移动效，只保留透明度变化。

## 实现拆分

建议分三步实现：

1. 新增 guide 状态、帮助按钮、首次引导弹窗。
2. 增加倒计时、流程词、编辑词库、导入导出的局部提示。
3. 优化动效和 prefers-reduced-motion，并补齐导入导出的 guide 状态兼容。

## 验收标准

- 首次打开能看到 5 步引导。
- 首次自动展示后，刷新页面不再自动出现首次引导。
- 点击帮助按钮可以重新查看完整引导。
- 点击“重新显示新手提示”后，首次引导再次出现。
- 点击倒计时后，倒计时提示不再出现。
- 完成任意流程词后，滑动提示不再出现。
- 打开编辑词库后出现编辑提示，关闭或操作后不再出现。
- 导入或导出成功后，导入导出提示不再出现。
- 页面仍然没有页面级滚动。
- `pnpm check` 和 `pnpm build` 通过。
